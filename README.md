## ğŸ§  Technical Details

### Memory Layout

The game is compiled as a flat `.com` binary (origin `0x0100`), meaning code and data share a single 64KB segment. All variables are stored inline in the data section.

### Interrupt Hooking

| Interrupt | Purpose |
|-----------|---------|
| `INT 8h` (IRQ0) | System timer â€” fires ~18.2 times/sec; controls snake speed via tick counter |
| `INT 9h` (IRQ1) | Keyboard â€” reads raw scancodes from port `0x60`; handles WASD |

Both original ISR vectors are saved and restored on exit to keep the system stable.

### Snake Movement Logic

Movement is entirely handled through direct video memory reads/writes:

1. The **head** writes a new cell with a color attribute encoding direction (`0x2F` + arrow char).
2. The **tail** reads its own cell's character to determine which way it should step next, then clears itself (`0x0720`).
3. **Collision detection** checks the destination cell's attribute â€” `0x2F_` = snake body = game over; `0x4C02` = apple = eat; `0x0720` = empty = move.

This eliminates the need for a separate snake position array.

### Speed Control

The timer ISR increments a `tick_count`. When it reaches the threshold (default: `2` ticks â‰ˆ ~9 ticks/sec), it sets a `move_snake` flag. The main game loop polls this flag. Lower the threshold value in `INT8_Handler` to increase speed.
